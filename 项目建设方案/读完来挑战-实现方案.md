# 读完来挑战 — 实现方案

## 背景

学习应用（学诗词/学国学/学通识）目前是只读浏览器，有"读"没"学"。在阅读器看完一节后弹出挑战题目，利用主动回忆提升学习效果。

## 命名

功能名：**「读完来挑战」**（避免"考"字，与"好大儿来挑战"品牌一致）
- 弹窗标题："来挑战一下！" / "练一练"
- 按钮文案："开始挑战" / "继续挑战"
- 结果文案："挑战完成！答对 X/Y 题"

## 核心原则

**题目必须和刚读完的内容严格对应。** 读完《静夜思》不能弹出杜甫的诗句题。

实现方式：每道题生成时记录其来源章节（`section_ref`），取题时按章节精确过滤。

## 整体架构

- **题库**：现有 tiaozhan game.db（新增理解题也写入同一数据库）
- **题目关联**：`quiz_questions` 表新增 `section_ref` 字段，记录题目来源
- **数据获取**：学习应用 fetch → tiaozhan `GET /api/quiz/solo`，传 `section_ref` 参数精确定位
- **触发时机**：阅读器返回时（与现有 `watch(currentView)` 上报进度的逻辑合并）
- **展现形式**：全屏弹窗 Modal，看完一节弹出 3-5 题
- **科目映射**：xueshici → `shici`，xueguoxue → `guoxue`，xuetongshi → `general`

## 实现步骤

### 步骤 1：数据库加 section_ref 字段

**文件**：`apps/tiaozhan/server/db.js`

`quiz_questions` 表新增一列：
```sql
ALTER TABLE quiz_questions ADD COLUMN section_ref TEXT;
```
或建表时一并加入：`section_ref TEXT DEFAULT ''`

`section_ref` 格式：
- 诗词：`shici:{poemId}:{sectionId}` — 如 `shici:1:0`
- 国学：`guoxue:{classicId}:{sectionId}` — 如 `guoxue:lunyu:1`
- 通识：`general:{topicId}:{sectionId}` — 如 `general:t1:0`

同诗词内"下一句""作者是谁""理解题"都关联到对应章节，确保读完哪节出哪节的题。

### 步骤 2：tiaozhan 服务端改造

**文件**：
- `apps/tiaozhan/server/index.js`
- `apps/tiaozhan/package.json`

1. 安装 `cors` 包，添加 `app.use(cors())` 允许跨域
2. `GET /api/quiz/solo` 新增参数：
   - `limit`：返回题数（当前硬编码 15），支持 `?limit=3`
   - `section_ref`：可选，有则只返回该章节题目，无则返回全科目随机（保持向后兼容）

查询逻辑：
```sql
-- 有 section_ref：
SELECT * FROM quiz_questions 
WHERE section_ref = ? 
ORDER BY RANDOM() LIMIT ?

-- 没有 section_ref（老行为）：
SELECT * FROM quiz_questions 
WHERE category IN (...) 
ORDER BY RANDOM() LIMIT ?
```

### 步骤 3：题库生成脚本改造（关键）

**文件**：`apps/tiaozhan/scripts/generate-questions.ts`

#### 3.1 现有题型补充 section_ref

**诗词**所有题型（下一句/作者/出自）在生成时记录所属章节：
```typescript
// 当前：只写 category
db.prepare(`INSERT INTO quiz_questions (category, question, options, answer, difficulty) VALUES (?,?,?,?,?)`)
  .run('shici', ...)

// 改为：加 section_ref
db.prepare(`INSERT INTO quiz_questions (category, question, options, answer, difficulty, section_ref) VALUES (?,?,?,?,?,?)`)
  .run('shici', ..., `shici:${poem.id}:${section.id}`)
```

**国学**所有题型同理，section_ref = `guoxue:${classic.id}:${section.id}`

**通识**所有题型同理，section_ref = `general:${topic.id}:${section.id}`

**注意**：同诗词的"作者是谁"题，尽管多个 section 共用一个作者，也每条记录对应 section_ref，这样不管用户读到哪节都能匹配到该诗的题。

#### 3.2 新增理解题题型

基于各科已有的 `interpretation` / `content` 字段生成：

**诗词理解题**（基于 section.interpretation 赏析字段）：
- **情感题**："《{title}》这首诗表达了怎样的情感？" — 从 interpretation 提取情感关键词（思乡/爱国/孤独/喜悦等），用同诗其他 section 或同作者其他 section 的情感描述做干扰项
- **主旨题**："《{title}》这首诗主要描写了什么？" — 从 interpretation 提取场景/主题
- **意象题**："诗中的「{keyword}」指的是什么？" — 从 interpretation 提取意象解释

**国学理解题**（基于 section.interpretation 解读字段）：
- **核心思想题**：'{原文名句}' 这句话的核心意思是什么？ — 从 interpretation 提取
- **道理题**："这段文字告诉我们什么道理？" — 从 interpretation 提取

**通识理解题**（基于 section.content 内容字段）：
- **正误判断**："根据上文，以下说法正确的是？" — 从 content 提取事实，修改部分细节做干扰项。三种错误类型：偷换数值、张冠李戴、逻辑颠倒

通识已有定义题/因果题，理解题作为补充，每节新增 1 道正误判断题。

预计新增 **1,000-2,000 道理解题**，全部带 section_ref。

### 步骤 4：共享组件 — ReadingChallenge.vue

**新建文件**：`apps/shared/src/components/ReadingChallenge.vue`

功能参数：
```
Props:
  visible: boolean         — 弹窗开关
  subject: string          — 科目 (shici/guoxue/general)
  sectionRef: string       — 当前阅读的章节标识
  onClose: () => void      — 关闭回调
```

交互流程：
1. 弹窗打开 → 立即 fetch `/api/quiz/solo?subjects={subject}&section_ref={sectionRef}&limit=3`
2. 加载中显示骨架屏
3. 每题展示：题干 + 4 个选项按钮
4. 点击选项 → 正确选项变绿，错误选项变红（选错时）→ 1.5 秒自动下一题
5. 最后展示结果页：得分 + "再来一组"（重新 fetch 同一章节的题）/ "返回" 按钮
6. 边界处理：
   - 该章节无题目 → 静默关闭，不弹窗
   - API 失败 → 静默关闭，不阻塞用户
   - 未登录 → 可正常答题，但不上报成绩

### 步骤 5：三个学习应用接入

**文件**：
- `apps/xueshici/src/App.vue`
- `apps/xueguoxue/src/App.vue`
- `apps/xuetongshi/src/App.vue`

改动：

1. 引入 ReadingChallenge 组件
2. 在 `watch(currentView)` 中，检测到离开 reader 时，记录当前章节的 `sectionRef`：
```typescript
// 新增变量
const showChallenge = ref(false)
const challengeSubject = ref('')
const challengeSectionRef = ref('')

// 在 watch 中，离开 reader 时触发（保留原有学习进度上报代码）
watch(currentView, (newView, oldView) => {
  if (oldView === 'reader' && newView !== 'reader') {
    // 原有进度上报代码不变
    // ...reportLearningProgress...
    
    // 新增：触发挑战弹窗
    if (challengeSectionRef.value) {
      showChallenge.value = true
    }
  }
  if (newView === 'reader') {
    // 记录本次阅读的章节
    challengeSectionRef.value = `${subject}:${itemId}:${sectionId}`
  }
})
```

3. 模板中插入 `<ReadingChallenge>`

### 步骤 6：重新构建 + 部署

1. 重新运行 `generate-questions.ts`（现有题型补充 section_ref + 新增理解题）
2. 构建三个学习应用（xueshici/xueguoxue/xuetongshi）
3. 重启 tiaozhan 服务
4. 部署到生产环境
