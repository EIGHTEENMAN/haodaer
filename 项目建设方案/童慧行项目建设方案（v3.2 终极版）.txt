第一部分 设计方案

一、项目概述

童慧行是一个专为儿童和父母打造的成长陪伴平台，以"寓教于乐"为核心理念，覆盖国学启蒙、英语学习、通识百科、亲子旅行、益智游戏等多个方向。

平台包含两大独立站点：
- 童慧行主站（grandand.com）：聚合国学启蒙、诗词学习、通识学习、英语学习、通识百科、益智游戏等子应用，以及统一认证服务
- 童慧行走天下（travel.grandand.com）：亲子旅行攻略分享平台，汇聚真实家庭的亲子出行经验

从国学经典学立身，到通识百科长见识，再到走向广阔天地——童慧行陪伴孩子们读万卷书，建立精神内核；行万里路，打开世界的格局。在知识与实践的循环中，完成知行合一的成长闭环。

两个站点共享统一的用户注册和认证体系，支持微信扫码（需绑定手机号）、手机号、用户名注册（需绑定手机号）三种注册方式。每个站点独立开发，按网页端、小程序、APP 顺序逐步扩展。

我们始终把儿童体验放在第一位。从隐私保护到视力呵护，从界面设计到内容筛选，每一个细节都以孩子的安全和健康为出发点。孩子说好才是真的好。

平台开发分为两个运营阶段：
- 自主运营阶段：系统自主运行，无人工运营人员。商城仅提供虚拟商品，内容审核全 AI 自动。
- 人工介入运营阶段：后续配置运营人员后，开启实物商城、商户系统、人工审核、客服等需要人工的功能。

    ├── 主站 H5 (main-site, port 3000)
    ├── 统一认证 (auth-service, port 3007)

2.13 用户行为分析与自动化改进闭环

本平台通过 PostHog Cloud 免费版实现用户行为数据采集与自动分析，形成"数据采集 → 自动报告 → 驱动改进"的闭环。

2.13.1 数据抓取层（埋点清单）

每个子应用的关键路径埋点如下：

主站 (main-site)：
- 首页各入口点击量（诗词、国学、英语、通识、挑战、论坛、商城）
- 注册流程漏斗：浏览注册页 → 选择方式 → 填写信息 → 验证通过
- 登录方式分布：微信扫码 vs 手机号 vs 用户名
- 搜索框输入关键词分布

学诗词 / 学国学 / 学通识 / 学英语（学习类应用）：
- 内容打开率：各分类/篇章被点击次数
- 单篇完成率：用户看到第几句/第几页离开
- 学习流程各步流失率：浏览 → 阅读 → 答题 → 完成
- 英语按 L1-L4 级别统计学习进度
- 收藏/分享行为统计
- 搜索关键词分布
- 各分类内容热度排名（辅助首页推荐排序）

童慧行来挑战（游戏）：
- 每日对局次数、参与人数
- 各模式（问答挑战 / 神兽方块）参与分布
- 平均对局时长
- 胜率分布
- 赛季活跃度趋势
- 匹配等待时间分布（辅助判断是否需要调整匹配算法）

亲子旅行攻略 (travel-guide)：
- 攻略详情页打开率、平均停留时长
- 攻略点击分布（哪些攻略被打开最多）
- 评论发布率（浏览→评论转化）
- 点赞/收藏数据
- 评分分布
- 攻略搜索关键词
- 问答区活跃数据
- 儿童互动模块使用率（足迹地图、画廊、挑战等）
- 勋章达成分布

论坛 (forum)：
- 每日发帖量、评论量
- 帖子浏览→评论转化率
- 各版块活跃度排名
- 热点帖子自动识别（Hot Score 趋势）
- 搜索关键词分析
- 私信使用率

商城 (store)：
- 虚拟商品浏览→兑换转化率
- 各商品兑换量排名
- 积分消耗分布
- 节假日折扣点击率

2.13.2 自动化分析报告

PostHog 自带以下自动化能力，无需额外开发：

每日自动报告：
- 昨日新增用户数、活跃用户数
- 各子应用访问量 Top3
- 注册转化率（环比）
- 异常指标预警（指标骤降/骤升）

每周末自动报告：
- 本周热门内容 Top10（全站所有内容类应用）
- 本周热门攻略 Top5
- 论坛热帖 Top5
- 各应用 7 日留存率
- 游戏参与度周报
- 学习应用各分类完成率排名

月度自动报告：
- MAU（月活）趋势
- 各子应用用户分布占比
- 用户画像（时段偏好、内容偏好）
- 积分消耗与虚拟商品兑换月报
- 本月建议优化项（基于数据自动生成）

关键预警（主动通知）：
- 注册转化率低于阈值 → 检查注册流程
- 单日活跃用户骤降 50%+ → 检查服务器/网络
- 内容完成率低于 10% → 内容质量或呈现方式需要优化
- 游戏匹配等待 > 30 秒 → 匹配算法或参与度不足
- 攻略/论坛发布量为零持续 3 天 → 冷启动种子数据不足

2.13.3 数据驱动改进清单（系统侧）

通过对上述数据的分析，系统可自动或由开发人员人工做出以下改进：

| 数据表现 | 分析结论 | 改进动作 |
|---------|---------|---------|
| 某诗词/单词打开率高 | 用户偏好该类内容 | 首页/推荐位优先展示 |
| 某内容完成率低（大量用户在中间离开） | 内容过长或枯燥 | 拆分章节、增加互动元素 |
| 论坛某版块活跃度持续走低 | 内容吸引力不足 | 调整版块定位或合并 |
| 攻略某分类点击率高但评论少 | 内容消费型用户多，互动意愿低 | 降低互动门槛（一键表情代替文字评论） |
| 游戏某模式参与度骤降 | 难度曲线不合理或缺乏新鲜感 | 调整难度或增加随机元素 |
| 兑换率高的虚拟商品集中在某几款 | 其他商品吸引力不足 | 调整商品定价或视觉效果 |
| 注册流程某步骤流失率高 | 该步骤有体验障碍 | 简化表单、优化验证码体验 |

2.13.4 数据驱动改进清单（运营侧）

自主运营阶段（只读参考，无法人工操作）：
- 监控仪表盘展示上述关键指标，观察趋势
- 系统自动生成优化建议（见上表），以通知形式推送给管理员（邮件）

人工介入阶段：
- 运营人员根据月报调整内容推荐策略
- 针对流失率高的内容做人工优化
- 运营活动效果量化评估（如节假日折扣带来的兑换量提升）
- A/B 测试：同一页面两个版本分别展示给不同用户，数据决定最终版本

2.13.5 隐私与合规说明（儿童平台特别要求）

- 所有埋点数据不可携带 PII（个人身份信息），仅记录匿名设备 ID + 事件行为
- 不采集儿童精确位置、通讯录、相册内容
- 13 岁以下用户（儿童模式）禁止 Session 录制
- PostHog Cloud 数据存储区域选择新加坡或美国（符合跨境数据传输要求）
- 隐私政策中明确告知使用 PostHog 进行行为分析
- 用户可在设置中开启/关闭"使用体验改进计划"
- 数据保留周期：原始事件保留 90 天，聚合报表永久保留
- PostHog Cloud 服务等级为免费版，无 SLA 保障，数据丢失不影响核心业务运行

7.2 各模块自动运行机制

auth-service：启动自动建表（SQLite WAL），定时清理过期验证码（每 5 分钟），定时清理过期积分（每月 1 日）
travel-guide：AI 审核自动放行/拦截，勋章自动检测颁发，排行榜每小时自动刷新
haodaer-tiaozhan：赛季每月 1 日自动重置，匹配队列自动运行，战区基于 IP 自动判定
forum：AI 审核自动放行/拦截，Hot Score 算法自动排序，自动处罚升级
store：兑换即发放（原子事务），限购由代码保证，节假日折扣由时间条件自动触发

7.3 系统自愈机制

PM2 级别：内存超限自动重启、异常退出自动重启
服务级别：60 秒健康检查，连续 3 次失败自动重启
数据库级别：SQLite WAL 崩溃恢复、PostgreSQL auto-vacuum
磁盘级别：>80% 自动清理 30 天前日志，>90% 停止非核心任务，>95% 告警通知

7.4 无人工异常处理流程

账号异常 → 忘记密码 → 手机验证 → 自助重置
AI 误封 → 申诉通知 → AI 自动复核 → 通过则解封
积分异常 → 查看流水 → 点击反馈 → 自动核对 → 系统错误则补发
非流程问题 → FAQ 页面（覆盖 90%）→ 邮箱联系 → 运维回复
