## 二、Logo 设计

### 2.1 设计原则

- **儿童友好**：圆润、温暖、不尖锐
- **识别性强**：简化到即使缩小到 favicon 也能认出来
- **适配多场景**：网站、移动端、印刷品、社交头像
- **可衍生**：各子应用可以基于主标做变体

### 2.2 方案方向（3 选 1）

#### 方向 A：星星/光芒标
- 概念来源：主站现在用的 emoji 🌟
- 寓意：知识之光，启明星，照亮成长之路
- 形态：一颗五角星，角部做圆润处理，融入书本翻页线条
- 优势：与现有认知一致，过渡平滑，适合做动态效果

#### 方向 B：书 + 足迹组合标
- 概念来源：直接呼应 slogan "读万卷书，行万里路"
- 形态：左侧是一本翻开的书（书页构成山形），右侧是两行小脚印（一大一小 = 亲子）
- 优势：故事性强，一眼传达品牌理念
- 风险：元素较多，简化难度大

#### 方向 C：字母 "g" + "a" 图形标
- 概念来源：GrandAnd 首字母
- 形态：小写字母 "g" 和 "a" 连笔，构成一个笑脸侧脸或太阳轮廓
- 优势：国际化、简洁、适合做动态
- 风险：中文用户理解门槛稍高

### 2.3 设计执行路线

**第一步：草图阶段**
- 设计师（或 AI 辅助）出 5-10 个概念草图
- 筛选 2-3 个方向进入精稿
- 输出：黑白线稿，识别性测试

**第二步：精稿阶段**
- 确定曲线、比例、间距
- 配色上标（搭配标准色 #2563EB）
- 输出：彩色矢量稿 + 单色稿

**第三步：规范输出**
- 标志标准制图（网格坐标）
- 横式组合（中英文左右排列）
- 竖式组合（中文在上，英文在下）
- 图标独立使用（无文字场景）
- 最小使用尺寸（印刷 15mm / 屏幕 32px）
- 安全空间（标志高度的 1/4）
- 错误使用示例（拉伸/变色/加边框/旋转等）

**第四步：子品牌衍生**
- 好大儿走天下 = 主标 + 旅行元素（小飞机/地球）
- 好大儿来挑战 = 主标 + 能量元素（闪电/奖杯）
- 好大儿社区 = 主标 + 对话气泡
- 好大儿商城 = 主标 + 购物袋/星星

**第五步：落地物输出格式**
- SVG（首选，所有网页场景）
- PNG：@1x (32px)、@2x (64px)、@3x (96px)
- ICO（浏览器 favicon）
- 源文件：Figma 或 Adobe Illustrator

## 四、字体规范

### 4.1 中文字体

| 层级 | 字体 | 字号 | 字重 |
|------|------|------|------|
| 大标题 | Noto Sans SC | 28-36px | Bold (700) |
| 中标题 | Noto Sans SC | 20-24px | Semibold (600) |
| 正文 | Noto Sans SC | 14-16px | Regular (400) / Medium (500) |
| 辅助文本 | Noto Sans SC | 12-13px | Regular (400) |
| **通配** | `PingFang SC, Microsoft YaHei, sans-serif` | — | 后备字体 |

### 4.2 英文/数字字体
- 首选：**Inter** 或 **Montserrat**
- 后备：`-apple-system, BlinkMacSystemFont, sans-serif`
- 应用范围：Logo 副标 "GrandAnd"、数据统计、代码片段

### 4.3 行高（儿童视力保护）
- 标题：1.3-1.4
- 正文：1.6-1.8
- 按钮/标签：1.2

## 五、品牌手册制作

### 5.1 手册结构与内容

品牌手册建议包含以下章节（约 20-30 页 PDF）：

| 章节 | 内容 | 页数 |
|------|------|------|
| **01 品牌故事** | 好大儿是谁、为什么做、核心理念、品牌使命与愿景 | 2-3p |
| **02 Logo 规范** | 标准标志、组合规范、安全空间、最小尺寸、错误示例 | 4-6p |
| **03 色彩规范** | 主色/辅助色/子应用色/中性色，各色值对照（HEX/RGB/CMYK/Pantone） | 3-4p |
| **04 字体规范** | 中英文字体、字号层级、行高、应用示例 | 2-3p |
| **05 应用场景** | 网站展示、移动端、社交媒体头像、名片、PPT 模板 | 4-6p |
| **06 视觉元素** | 辅助图形、图标风格、插画风格、动效规范 | 2-3p |
| **07 品牌语调** | 文案风格指引（温暖亲切但不幼稚）、常用词汇、禁忌用语 | 2p |

### 5.2 制作工具

- **排版设计**：Figma（在线协作 + 导出 PDF）或 Canva（快速出稿）
- **打印输出**：导出 PDF（CMYK，300dpi，A4 尺寸）
- **在线版本**：在好大儿主站部署一个 `/brand` 页面（HTML 展示）

### 5.3 制作流程

| 步骤 | 工作内容 | 产出 |
|------|----------|------|
| 1 | 品牌体系内容定稿 | 以上 1-4 节内容确认 |
| 2 | Logo 设计完成 | SVG 矢量文件 + 规范尺寸 |
| 3 | 收集应用场景截图 | 各子应用关键页面截图 |
| 4 | Figma/Canva 排版设计 | 手册初稿 |
| 5 | 内部审校 | 反馈修改 |
| 6 | 输出正式版 | PDF + 在线版 |

### 5.4 维护管理

- 品牌手册应为"活文档"，随品牌发展迭代
- 每次迭代更新版本号（v1.0 → v1.1 → v2.0）
- 变更记录集中在手册末尾更新日志
- 挂载位置：`grandand.com/brand` 或内部共享文件夹

## 六、实施路径

### Phase 1：品牌基础统一（耗时约 2 天，零成本）
- 确定主选 Logo 方向（从设计方向 A/B/C 中选定一个）
- 确定子应用色彩体系（已有，只需确认）
- 确定字体方案（已有，只需确认）
- 输出：本方案定稿确认

### Phase 2：Logo 设计执行（耗时约 3-5 天）
- AI 辅助出图
- 草图 → 精稿 → 规范输出 → 子品牌衍生 → 落地格式
- 输出：SVG/PNG/ICO 全套 + 使用规范

### Phase 3：品牌手册制作（耗时约 3-5 天）
- Figma 排版设计（简洁商务风）
- 内容撰写 + 截图收集
- 输出：PDF 手册 + 在线页面

### Phase 4：视觉落地（后续按需推进）
- 各应用按品牌规范更新 UI
- 共享组件库全面建设
