# 学英语板块改进方案

> 制定时间：2026-05-25 | 最后更新：2026-05-26
> 基于用户需求讨论整理
>
> ✅ = 已完成 | 🔄 = 部分完成/开发中 | ⬜ = 待开发

---

## 一、核心目标

**不做创新，孩子喜欢就好。** 让孩子在玩的过程中自然记住英语单词，还能和好朋友在线组队一起玩、互相喊话。

### 四条标准

1. 孩子愿意打开玩
2. 玩得开心
3. 顺带学了单词
4. 能和队友互动

---

## 二、当前问题

| # | 问题 | 现状 |
|---|------|------|
| 1 | 主题画面不匹配 | 所有世界共用同一个竞技场布局，只是背景颜色不同。ANIMAL 看起来不像丛林，FOOD 看起来不像厨房 |
| 2 | 单词没有由易到难 | 一个世界前5关都是 `difficulty=1`，词汇没区分度。打到第6关 BOSS 才升到2，但已经打了5关同样的难度了 |
| 3 | 造型太简单 | 角色是像素方块拼的（Graphics API fillRect），没有圆润感、没有表情、没有动作帧。特效是手写矩形/圆形粒子，不够绚 |
| 4 | 玩法单一 | 全程"进场 → 学词 → 打怪 → 下一关"，缺少变化。只能一个人玩，不能和朋友一起 |
| 5 | 没有社交动力 | 单人游戏，孩子的朋友不能加入，缺少"一起玩"的吸引力 |

---

## 三、改进方案总览

四个改进方向，按优先级排列：

```
第一刀 → 画面升级：每个世界有主题特色，角色更可爱，特效更炫
第二刀 → 难度渐进：每关难度递增，而不是5关同一个难度
第三刀 → 在线组队：两个孩子各自设备上组队一起打怪
第四刀 → 流程微调：缩短进入战斗的路径，Boss更有仪式感
```

---

## 四、第一刀：画面升级（让每个世界有"世界感"）

### 4.1 主题背景

每个世界用 Phaser Graphics API 程序化绘制独特的背景画面，不依赖外部图片资源。

| 世界 | 背景方案 |
|------|---------|
| **ANIMAL** 动物 | 绿色草地（下半）+ 蓝色天空（上半）+ 树桩/草丛装饰 |
| **FOOD** 食物 | 暖色台面 + 锅/碗/勺轮廓装饰 |
| **COLOR** 颜色 | 彩虹色渐变背景 + 几何形状漂浮 |
| **BODY** 身体 | 粉色/肤色背景 + 圆形/椭圆装饰 |
| **CLOTHES** 衣服 | 紫色/深蓝背景 + 衣架/纽扣装饰 |
| **NATURE** 自然 | 蓝白渐变天空 + 云朵/雨滴/太阳装饰 |
| **DAILY** 日常 | 米色室内墙 + 窗户/门/家具轮廓 |
| **NUMBER** 数字 | 深蓝星空 + 数字/时钟轮廓漂浮 |
| **FAMILY** 家庭学校 | 暖黄教室背景 + 黑板/书架轮廓 |
| **ACTION** 动作 | 运动场绿草地 + 跑道线/球类装饰 |

实现方式：在 `GameScene.ts` 的 `create()` 中，读取当前世界 ID，调用对应的背景绘制函数。本质就是画几个渐变色矩形 + 几个装饰形状。

### 4.2 地面纹理

当前是 80px 网格线，改成每个世界不同颜色的柔和纹理线。

### 4.3 障碍物

从纯色方块改成与主题相关的形状（同样是 Graphics API 绘制）：

| 世界 | 障碍物样式 |
|------|-----------|
| ANIMAL | 灌木丛（绿色圆弧顶） |
| FOOD | 锅（半圆+手柄） |
| COLOR | 彩色方块 |
| BODY | 圆角药丸 |
| CLOTHES | 衣架 |
| NATURE | 石块 |
| DAILY | 椅子 |
| NUMBER | 数字方块 |
| FAMILY | 书桌 |
| ACTION | 跨栏 |

### 4.4 角色造型升级

**玩家角色**（Player.ts）：
- 当前：54×66 像素方块人（黄衣服 Bruce Lee）
- 改为：头大身小的 Q 版造型，圆角矩形，颜色明亮
- 手上拿一把小枪（程序化绘制，圆润卡通风格，不可怕）
- 加简单表情（高兴时笑脸、射击时专注表情、受伤时皱眉、换弹时紧张表情）
- 移动时身体上下浮动（sine 缓动，幅度 3px）
- 射击时枪口闪光 + 身体后坐力（后仰再回弹）
- 换弹时低头 + 手上动作

**怪物造型**（Monster.ts）：
- 当前：4种身体类型的像素块
- 改为：按世界主题区分基本轮廓（ANIMAL→四足动物轮廓、FOOD→圆形食物轮廓、NATURE→云/水滴等）
- 颜色明亮，加眼睛和表情
- 被击中时：缩放弹动 + 闪白

### 4.5 特效增强

> 不追求 AAA 品质，目标是"孩子看了哇哦一声"

| 场景 | 当前效果 | 改进效果 |
|------|---------|---------|
| 打中怪物 | 金色环 + 方形粒子 | 彩色星形/心形粒子爆发 + 缩放弹动 + 闪光 |
| 高连击 (5+) | 屏幕边缘发光 | 同上 + 连击数字放大弹出 + 屏幕闪烁 |
| 技能释放 | 对应颜色的环+粒子 | 冲击波扩散 + 全屏闪色 + 粒子数量翻倍 |
| BOSS 出场 | 无特殊效果 | 屏幕暗下来 + BOSS 名大字显示 +小震动 |
| 过关 | "STAGE CLEAR!" 文字弹出 | 彩色烟花 + 文字放大弹入 + 背景光效 |
| 打错了 | 小火花 | 角色出现问号表情，无惩罚性反馈 |

## 五、第二刀：难度渐进安排

### 5.1 当前问题

以 ANIMAL 世界为例：
- Stage 1~5：全部 `wordDifficulty: 1`（都是最简单的词）
- Stage 6：`wordDifficulty: 2`（唯一变化）

这样前5关打下来词汇难度没变化，感觉不到进步。

### 5.2 改进方案

每关的 wordDifficulty 平滑递增，同时利用已有的 offset 机制避免重复。

**ANIMAL 世界改造后（示例）：**

| 关卡 | 当前 | 改为 | 说明 |
|------|------|------|------|
| Stage 1 | diff=1 | diff=1 | 入门词：cat, dog, fish... |
| Stage 2 | diff=1 | diff=1 | 同难度换一批词（offset机制已有） |
| Stage 3 | diff=1 | diff=1~2 | 混入少量稍难词：turtle, parrot... |
| Stage 4 | diff=1 | diff=2 | 大部分中等难度 |
| Stage 5 | diff=1 | diff=2~3 | 加入较难的词 |
| Stage 6 BOSS | diff=2 | diff=1~3 | 混合所有已学词复习 |

**改动范围**：只需修改 `apps/english/src/data/stages.ts` 中每个世界的 stages 数组的 `wordDifficulty` 值，代码逻辑无需改动。

### 5.3 跨世界难度

当前已按 Grade 分级（初级→高级），保持不变，但让 Grade 内部的世界也有难度递增：

| Grade | 世界1 | 世界2 |
|-------|-------|-------|
| 初级 | ANIMAL (diff 1-3) | FOOD (diff 2-4) |
| 基础 | COLOR (diff 2-4) | BODY (diff 3-5) |
| 中级 | CLOTHES (diff 3-5) | NATURE (diff 4-6) |
| 中高级 | DAILY (diff 4-6) | NUMBER (diff 5-7) |
| 高级 | FAMILY (diff 5-7) | ACTION (diff 5-7) |

这样孩子每解锁一个新世界，起点难度就是上一个世界的终点附近，形成平滑过渡。

## 六、第三刀：在线组队玩法

### 6.1 核心思路

**两个孩子在各自设备上组队，进入同一关卡，一起打怪。**

与单人模式并行不悖——不想组队就单人玩，和现在一样。

### 6.2 参考现有基础设施

tiaozhan（挑战）已有一整套 WebSocket 多人联机系统：

| 组件 | tiaozhan 实现 | 复用方式 |
|------|-------------|---------|
| WebSocket 服务器 | `ws` 库，端口 3001 | 复制模式到 english |
| JWT 认证 | URL query `?token=` 参数 | 直接复用 |
| 房间码系统 | 4 位字母数字 | 直接复用 |
| 客户端 WebSocket | React 版 | 重写为 TS 版 |
| 消息协议 | JSON `{type, ...}` | 精简改造 |

### 6.3 新增服务端文件

```
apps/english/
  server/
    index.js              # Express + WebSocketServer
    games.js              # 游戏房间管理 + 状态同步
  src/
    game/multiplayer/
      network.ts          # WebSocket 客户端
      coop-room.ts        # 组队界面（创建/加入房间）
```

### 6.4 游戏流程（组队模式）

```
单人模式入口                   组队模式入口
  │                              │
  │                         显示房间码
  │                              │
  │      另一玩家输入房间码加入────┘
  │                              │
  │                         双方确认
  │                              │
  └──── 进入同一个关卡 ──────────┘
             │
         共享怪物波次
             │
      两个角色同时打怪
             │
        通关 / 失败
```

### 6.5 组队玩法细节

- 两个角色在同一战场出现（绿名 vs 蓝名区分）
- 怪物血条共享——谁打到都扣血
- 所见即所得：队友的位置、朝向、状态（正在打哪个怪）实时同步
- 如果一方掉线，怪物数量和难度自动降低，另一方继续

### 6.6 自然互动——快捷语按钮

在屏幕边缘放几个快捷语按钮，孩子可以快速点击跟队友"说话"：

```
[我来打这个！]  [好棒！]  [帮帮我！]
[这个我会！]    [Boss 来了！]  [一起上！]
```

**为什么用快捷语而不是自由输入或语音：**
- 孩子打字慢，快捷语零延迟
- 避免恶意发言，快捷语都是正面/合作导向
- 快捷语本身也是英语学习——按钮显示英文（"My turn!" "Great!" "Help!"）

### 6.7 消息协议

复用 tiaozhan 的 JSON 消息模式，精简为组队所需的最小集：

**客户端 → 服务器：**

| 类型 | 说明 | 数据 |
|------|------|------|
| `create_room` | 创建组队房间 | 世界ID, 关卡号 |
| `join_room` | 加入房间 | 房间码 |
| `ready` | 准备就绪 | - |
| `player_move` | 角色移动 | x, y, direction |
| `player_attack` | 攻击 | targetId, skillType |
| `chat` | 快捷语 | messageId |

**服务器 → 客户端：**

| 类型 | 说明 | 数据 |
|------|------|------|
| `connected` | 连接成功 | userId |
| `room_created` | 房间创建成功 | roomCode |
| `room_update` | 玩家状态变更 | players[] |
| `game_start` | 游戏开始 | stageConfig |
| `monster_update` | 怪物状态 | monsters[] |
| `player_update` | 对方位置/动作 | playerData |
| `monster_damaged` | 怪物受伤 | monsterId, hp, attackerId |
| `monster_die` | 怪物死亡 | monsterId, killerId |
| `stage_clear` | 过关 | results |
| `game_over` | 失败 | - |
| `chat` | 对方发了快捷语 | message |

### 6.8 同步策略

- 不追求帧同步（开发成本高、延迟敏感）
- 采用服务器权威 + 状态快照模式
- 服务端每 100ms 广播一次全局状态
- 客户端插值平滑位置
- 仅同步关键状态：角色位置、怪物HP、攻击事件

## 七、宝可梦化玩法设计

### 7.1 核心设计原则（基于二语习得研究）

| 研究结论 | 游戏设计原则 |
|---------|------------|
| 语言是"泡"出来的，不是"学"出来的 | 游戏用图标就能玩通，不要求懂英语，但英语出现在每个自然场景中 |
| 情境化学习最有效 | 每个单词对应一个单词兽造型，ANIMAL世界学动物词，FOOD世界学食物词 |
| 高频重复 + 不同情境 | 同一个词在遭遇、战斗、捕捉、图鉴、进化中反复出现，每次情境不同 |
| 不强迫输出（尊重静默期） | 所有英语互动都是可选的，不存在"不懂英语就卡关" |
| i+1 可理解性输入 | 单词从易到难分布，每个世界内从简单词开始，逐步进阶 |
| 动机驱动 > 任务驱动 | "集齐图鉴"的内在欲望让孩子主动反复玩 |
| 社交催化语言习得 | 交换、对战、比收集进度 → 孩子之间自然谈论单词 |

### 7.2 英语在游戏中的三层分布

**第一层：核心游玩层（不看英语也能玩）**

| 环节 | 英语出现方式 | 孩子怎么玩 |
|------|------------|-----------|
| 在世界里走路 | 地图上显示 "ANIMAL WORLD" | 看画面就知道在哪，不需要读字 |
| 遇到野生单词兽 | 怪物身上显示英文名（CAT） | 看造型就知道是什么动物 |
| 战斗选指令 | 图标为主（拳头=攻击、袋子=道具） | 看图就能选，不需要读字 |
| 捕捉 | 点击精灵球图标 | 图标就是精灵球，一看就懂 |
| 图鉴 | 单词 + 图片 + 中文 | 看图片就知道是什么词 |

**第二层：自然暴露层（不刻意但每次都在看）**

| 场景 | 看到的英语 | 频次 |
|------|-----------|------|
| 战斗中 | 对方名字 "DOG" 一直显示在屏幕上方 | 每场战斗几十秒 |
| 指令按钮 | "FIGHT" "ITEM" "RUN" 配图标 | 每场选好几次 |
| HP 条 | "HP" + 血量数字 | 每回合都在看 |
| 背包 | "Potion" "Candy" "Poke Ball" 配图标 | 每次打开背包 |
| 捕捉成功 | "CAUGHT! DOG was added!" + 闪过卡片 | 每次捕捉 |
| 进入新区域 | "Welcome to FOOD WORLD!" | 每次切换世界 |
| 进化 | "EVOLUTION! → CAT evolved into KITTEN!" | 每次进化 |

**第三层：主动深入层（想学更多的孩子自己探索）**

| 环节 | 做什么 | 学到什么 |
|------|-------|---------|
| 图鉴详情 | 点开单词卡看详情 | 读音、例句、拼写 |
| 战斗中专注 | 选"FOCUS"进行单词确认（可选，答对伤害加倍） | 加深单词印象 |
| 收集进度 | 查看"已收集 45/300" | 自然复习已收集的词 |
| 和朋友交换 | 对话"我拿 CAT 换你的 DOG" | 说出单词名 |

### 7.3 游戏基本流程

```
选择世界（10个主题）
     ↓
进入野外地图（俯视角，像宝可梦那样走路）
     ↓ ← 野生的单词兽出现（身上显示英文单词）
进入战斗（回合制）
     ↓ ← 选择指令：FIGHT / ITEM / CAPTURE / RUN
削弱野生单词兽（战斗自动进行，不需要英语）
     ↓
投掷精灵球捕捉
     ↓
捕捉成功！→ 弹出单词卡（词+图+中文）
     ↓
加入图鉴（已收集 X/300）
     ↓
培养单词兽：战斗升级 → 进化
        或：和朋友交换/对战
```

### 7.4 战斗系统

回合制，像宝可梦但简化。

**战斗画面布局：**

```
┌──────────────────────────────────────┐
│  你的 CAT  🐱    野生的 DOG  🐶      │  ← 英语单词自动显示
│  LV.3  HP ██████  15/15             │     "(野生DOG出现了！)"
│         LV.2  HP ██████  12/12      │
│                                      │
│                                      │
│                                      │
│  ┌──────────────────────────────┐    │
│  │ [👊 FIGHT]  [⭐ FOCUS]      │    │  ← 图标为主，英文辅助
│  │ [🔁 SWITCH] [🏃 FLEE]       │    │  ← SWITCH=换精灵上场
│  └──────────────────────────────┘    │
└──────────────────────────────────────┘
```

**指令说明：**

| 指令 | 图标 | 效果 | 需要英语吗 |
|------|------|------|-----------|
| FIGHT | 👊 | 自动攻击，看攻击动画 | ❌ 图标就懂 |
| SWITCH | 🔁 | 换另一只单词兽上场 | ❌ 图标就懂 |
| FLEE | 🏃 | 逃跑 | ❌ 图标就懂 |
| FOCUS | ⭐ | 进入单词确认模式（答对伤害翻倍） | ✅ **可选** |

> BAG（道具）暂不实现，未来扩展。

### 7.4.1 队伍与切换（SWITCH）系统

**这是处理"战斗输了怎么办"的核心机制，完全参考宝可梦。**

#### 队伍组成

```
进入战斗时，系统自动组建队伍：
  1. 从图鉴中已捕捉的单词兽中选取最多 6 只
  2. 排序优先级：进化阶段高 → 等级高 → 最近捕获
  3. 第1只自动上场（当前战斗单词兽）
  4. 如果图鉴为空，使用系统默认单词兽（基础 CAT）
```

| 队伍槽位 | 来源 | 说明 |
|---------|------|------|
| 第1位 | 当前战斗的单词兽 | 系统默认或上次战斗用过的 |
| 第2-6位 | 图鉴已捕捉 | 按优先级自动填充 |
| 空位 | — | 不足6只时显示为"空" |

#### SWITCH 机制

```
战斗任何时候 → 按 SWITCH → 弹出队伍列表
  ├── 正常切换：当前下台 → 选中的上场（消耗当前回合）
  ├── 倒下自动切：当前 HP=0 → 自动弹出队伍列表
  │     （不消耗回合，但需选一只上场）
  └── 全部倒下：全员 HP=0 → 战败流程
```

**切换规则：**
- 不能选已经上场的
- 不能选 HP=0 的（倒下的）
- 可以选择同一只（取消切换，回到战斗）
- 切换时显示每只单词兽的 HP 状态

#### 战败处理

```
全员 HP=0 → 显示战败画面
  ├── 显示信息："全员倒下了！回到营地休息..."
  ├── 回到 Overworld
  ├── 自动全员满血复活（零惩罚）
  └── 错过的单词兽消失（需要重新遭遇）
```

**核心原则：** 对小孩零惩罚。打输只是没抓到这个单词而已，回满血继续玩。

#### 战斗流程更新

```
开始战斗
  ├── 队伍组建 → 第1只上场
  ├── 显示野怪信息
  │
  └── 回合循环：
        ┌─ 玩家回合 ──────────────────┐
        │  FIGHT → 攻击野怪            │
        │  FOCUS → 单词确认（可选）    │
        │  SWITCH → 弹出队伍选单       │
        │  FLEE → 逃跑，回到地图       │
        └─────────────────────────────┘
               ↓
        ┌─ 野怪回合 ──────────────────┐
        │  野怪反击 → 扣我方 HP       │
        └─────────────────────────────┘
               ↓
        ┌─ 战后判断 ──────────────────┐
        │  野怪 HP=0 → 可捕捉         │
        │  我方 HP=0 → 自动弹队伍选单  │
        │  全员 HP=0 → 战败回地图      │
        └─────────────────────────────┘
```

**FOCUS 机制（唯一的英语互动）：**
- 战斗中按 FOCUS → 进入单词确认
- 屏幕显示当前目标单词（如 DOG）
- 4 个选项（图或中文），选对 → "Correct! Damage x2!" → 这回合伤害翻倍
- 选错 → 没加成，正常攻击（无惩罚）
- **可以不按 FOCUS，直接 FIGHT 正常打**

### 7.5 捕捉系统

```
野生的 DOG HP 降到很低了！
→ 出现 [投掷精灵球] 按钮（图标是精灵球，一看就懂）
→ 球摇晃... 摇晃... 叮！
→ 捕捉成功！

弹出单词卡：
┌─────────────────────────┐
│                         │
│       🐶 DOG            │  ← 大号英语单词
│       狗                │  ← 中文释义
│       /dɒɡ/             │  ← 发音
│       "I have a dog."   │  ← 例句
│                         │
│       ★ 已加入图鉴 ★    │
│     目前收集: 45/300     │
└─────────────────────────┘

单词卡自动展示 3 秒后消失。孩子可以多看几眼，也可以点掉。
```

### 7.6 图鉴与收集系统

**这是游戏的核心驱动力。**

- 每个世界 300 个单词 = 300 只单词兽
- 图鉴以网格展示所有已捕获的单词兽
- 未捕获的显示剪影（激发收集欲）

**图鉴页面：**
```
┌──────────────────────────────────────┐
│  ★ 图鉴 - ANIMAL WORLD ★            │
│  已收集 45 / 300                     │  ← 进度条
│                                      │
│  🐱  🐶  🐰  🐟  🐦  🐸  🦁  🐯   │  ← 已捕获的
│  CAT  DOG  RAB  FIS  BIR  FRO  LIO  TIG│
│                                      │
│  ❓   ❓   ❓   ❓                   │  ← 未捕获的（剪影）
│  ???  ???  ???  ???                  │
│                                      │
│  稀有度: ★常见 ★★稀有 ★★★史诗     │
└──────────────────────────────────────┘
```

**点击已捕获的单词兽 → 查看详情页：**
- 大图（程序化绘制的卡通造型）
- 英文单词
- 中文释义
- 音标 + 发音按钮
- 例句 + 中文翻译
- 进化阶段（当前/下一阶段）
- 捕获时间

### 7.7 进化系统

每只单词兽有 3 个进化阶段，进化 = 孩子对这个词的掌握程度提升。

| 阶段 | 名称 | 解锁条件 | 造型变化 | 战斗能力 |
|------|------|---------|---------|---------|
| 第1阶段 | 基础形态 | 刚捕获 | 小只、简单 | 基础攻击 |
| 第2阶段 | 进化形态 | 答对该词相关题目 5 次 | 更大、更精致 | 攻击力 +50% |
| 第3阶段 | 终极形态 | 答对该词相关题目 15 次 | 最大、最酷炫 | 攻击力 +100% |

**进化流程：**
```
条件满足时 → 屏幕闪白光 → "EVOLUTION!" 大字动画
→ CAT evolved into KITTEN! → 单词兽造型变化
→ 新能力解锁！→ 孩子超有成就感
```

**进化的隐藏价值：**
- 孩子为了让自己喜欢的单词兽进化，会主动反复使用/确认这个词
- 每答对一次都在加深记忆
- 15 次答对 ≈ 这个词已经牢牢记住了

道具从战斗胜利、探索拾取、完成收集目标等途径获得。

### 7.9 社交系统

**交换：**
- 两个孩子在线连接，各自选一只单词兽交换
- 交换后双方的图鉴都增加对方给的词
- 自然对话："我拿 CAT 换你的 DOG 好不好？"

**对战：**
- 两个孩子用各自收集的单词兽对战
- 回合制，和打野生怪一样
- 赢了的得经验糖果

**图鉴比拼：**
- 可以看到朋友的收集进度
- "你才 50 个，我已经 80 个了！" → 好胜心驱动

## 八、不做的事情

- ❌ 不做实时动作战斗——改为回合制宝可梦式对战，更适合5-10岁
- ❌ 不搞射击/弹幕——回合制+图鉴收集为核心玩法
- ❌ 不强迫孩子输出英语——FOCUS等英语互动均为可选，不存在"不懂英语就卡关"
- ❌ 不用外部美术素材——全部用 Phaser 内置 Graphics API 程序化绘制
- ❌ 不做语音聊天——快捷语 + 自然交流就够了
- ❌ 不推翻现有 Vue 3 + Phaser 4 架构——在现有代码上增量修改
- ❌ 不加抽卡/内购——儿童产品，非商业化优先
- ❌ 不设限时/扣分惩罚——答错了无惩罚，孩子不会挫败

## 十、开发顺序建议

```
第一轮（核心玩法）
├── 野外探索场景（俯视角走路、遇到单词兽）
├── 回合制战斗（FIGHT/BAG/SWITCH/FLEE/FOCUS）
├── 捕捉动画 + 单词卡展示
└── 图鉴基础功能（查看已收集的单词兽）

第二轮（完整玩法）
├── 10个世界地图、每个世界不同单词兽
├── 进化系统（答对5次→进化、15次→终极）
├── FOCUS 可选英语互动
├── 道具系统（Potion、Poke Ball等）
└── 背包界面

第三轮（社交功能）
├── 在线交换房间（WebSocket 服务器）
├── 交换流程（选单词兽 → 确认交换）
├── 好友对战（回合制PVP）
└── 图鉴比拼

第四轮（打磨）
├── 精美动画（进化、捕捉、技能特效）
├── 主题画面升级（10个世界的视觉差异）
├── 难度调试
└── 性能优化
```

## 十一、验证方式

1. 进入游戏 → 世界选择 → 进入 ANIMAL 世界
2. 在野外地图走路 → 遇到野生单词兽（如出现"CAT"造型）
3. 进入战斗 → 看到 FIGHT/BAG/RUN 指令 → 选 FIGHT 自动攻击
4. 选 FOCUS → 出现单词选择题（不看中文能不能选对？）
5. 攻击到残血 → 投掷精灵球 → 捕捉成功动画
6. 弹出单词卡 → 查看图鉴 → CAT 出现在列表中
7. 用 CAT 战斗 5 次 → 进化触发 → "EVOLUTION!" 动画
8. 和朋友交换：开两个窗口 → 进交换房间 → 互选单词兽交换
9. 一个世界集齐 50 个 → 感受"还想继续收集"的驱动力
